Hybrid Search (Vector + BM25) Neo4j has had native **Apache Lucene full-text indexes** (which use the BM25 algorithm) built into its database for years. To make this seamless for AI, the `neo4j-graphrag` package includes a specific `HybridRetriever` class. When you configure this retriever, you point it to both your Vector Index and your Full-Text Index. When a user searches for *"Needs AWS Solutions Architect Certification"*, the `HybridRetriever` will: 1.  Run the semantic Vector Search. 2.  Run the BM25 exact-keyword search. 3.  Automatically normalize the scores from both lists, merge them, and return the combined top candidates.
Cross-Encoder Re-ranking Re-ranking is typically handled at the **orchestration layer** (your Python backend), not inside the database engine itself. Neo4j fits perfectly into a Re-ranking pipeline. You would use Neo4j's `HybridRetriever` to pull the top 50 employee profiles. Then, you pass those 50 profiles into a Cross-Encoder model (either a local open-source HuggingFace cross-encoder or an API like Cohere Rerank). The Re-ranker evaluates those specific 50 documents against the job description and re-orders them with extreme precision.
